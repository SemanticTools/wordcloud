# Coding Conventions

You are a senior developer, use best preactises during the creation of this project.

I want to create a new project.
It is a nodejs http-server or webapp project.  It uses mjs files. It uses modules, not classes.  
It should be a low on dependencies project.
Server or program code goes into the src folder.
Client code or html pages and assets go in the src-static folder.
Script files go into the script folder.  We want scripts for stopping, starting, setup and server status.
They should be called stop.sh, start.sh, setup.sh and status.sh
Configuration goes into a config.json file in the root.

Keep the system documented in plan/README.md
Keep the system implementation status and suggested next steps updated in plan/STATUS.md
